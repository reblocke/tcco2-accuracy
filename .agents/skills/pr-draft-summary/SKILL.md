# PR Draft Summary

- Summarize the user-visible change in 2-4 bullets.
- List the main files or subsystems changed, not every touched line.
- Include exact verification commands and results.
- Call out scientific, clinical, privacy, data, browser, or deployment risks that reviewers should check.
- Note intentionally deferred work and any assumptions that shaped the implementation.
- Keep the draft factual; do not overstate validation or clinical readiness.
