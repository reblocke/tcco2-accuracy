# Public Copy Safety Review

- Check that public wording matches validated functionality and does not overclaim accuracy, clinical readiness, or regulatory status.
- Keep examples synthetic unless a cited public source is explicitly intended.
- Make uncertainty, scope, and limitations clear without cluttering the primary workflow.
- Use consistent terminology across README, docs, UI, and PR text.
- Treat copy changes that alter interpretation as behavior changes and verify docs/tests accordingly.
