# Code Change Verification

- Start from the command documented in `AGENTS.md`, `Makefile`, CI, or README.
- Run the narrowest meaningful check first, then the repo-level verification target when the blast radius is broad.
- For Python package changes, run unit tests and Ruff checks.
- For browser or staging changes, run the staging command and E2E or smoke tests.
- Report exact commands, pass/fail outcome, skipped tests, warnings, and any files generated by verification.
- Do not claim a command passed unless it actually ran.
